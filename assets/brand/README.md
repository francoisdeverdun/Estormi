# Estormi Brand Assets

Estormi is a private local memory system: personal sources are ingested locally, embedded, indexed, and connected later as durable memory traces.

Visual semantics (palette, typography, motifs) are canonical in `docs/design-system.md`; this folder owns the artwork files and their packaging.

Brand line:

> Ars Memoriae

Tagline:

> A private place for everything your life remembers.

The logo uses four ideas:

- **Illuminated initial `E`** (the blocked mark): a gold-gradient Cinzel majuscule on a **burgundy** ground inside a single gold keyline — a bold, modern take on a manuscript lettrine that holds up at any size. This is the brand mark, shared by the app icon and the masthead. It is the same artwork on macOS (`EstormiLogoMark` in `packages/ui-kit`) and iOS (`Brand.swift`), and the same device — with the section's own initial — serves as the in-content lettrine on section headers.
- **Wordmark**: `STORMI` in Cinzel beside the `E` mark (together: ESTORMI) — cream caps, generous letter-spacing — readable at any size.
- **Illuminated rule**: a hairline that fades at both ends with a centred burgundy lozenge in a gold frame and flanking quatrefoils — geometry and air, not a crowded vine — for quiet spacing between page zones. The two ornaments are reusable primitives in `packages/ui-kit`: the lozenge is `Diamond`, the four-petal quatrefoil is `Fleuron`. On iOS only `Fleuron` is mirrored, in `apps/estormi-ios/Sources/Design/Ornaments/Fleuron.swift`.
- **Modern dark product surface**: the app uses a flat charbon ink interface so the detail stays precise instead of decorative.

Primary files (canonical Estormi brand assets):

- `estormi-mark.svg` — the blocked illuminated `E` mark (burgundy ground + gold keyline). Carries its own ground, so it reads on any surface.
- `estormi-mark-dark.svg` — same mark, kept as the explicit dark-surface alias.
- `estormi-app-icon.png` — macOS app-icon source: burgundy ground, gold keyline, centred gold `E` (rounded to the iOS squircle). Fed through `scripts/generate_icons.sh` → `apps/estormi-macos/icons/`.
- `estormi-wordmark-dark.svg` — `ESTORMI` in gold-gradient Cinzel caps for dark surfaces.
- `estormi-wordmark-light.svg` — the same wordmark in deeper antique gold for light surfaces.
- `estormi-lockup.svg` — README and brand lockup: the `E` mark beside the STORMI wordmark, garlanded `Ars Memoriae`, and the illuminated rule — centred, mirroring the in-app masthead (`EstormiMasthead` in `packages/ui-kit` and `apps/estormi-ios/Sources/Design/Brand.swift`). Carries its own charbon ground, so it reads on any surface.
- `estormi-divider.svg` — the illuminated rule as a reusable separator (dark surfaces).
- `estormi-divider-light.svg` — the illuminated rule in deeper gold for light surfaces.
- `estormi-cap-<L>-{dark,light}.svg` — drop-cap lettrines: a solid-gold Cinzel capital matching the briefing's `::first-letter` rule (`packages/ui-kit/src/briefing.css`). Floated with `align="left"` to open a doc with a wrap-around illuminated initial; the `dark` tone is the briefing's bright `--or-clair`, the `light` tone a deeper gold for white backgrounds, paired via `<picture>`. Generated by `scripts/generate_caps.py` (currently `E`, `T`).
- `estormi-mark-32.png` — 32px raster of the blocked mark, for favicons and small icon slots.
- `estormi-illumination-chart.md` — source notes for the medieval-minimal system.

## Brand asset map (intentional split — do not consolidate blindly)

Brand assets live in six roots on purpose; each is served or bundled by a
different mechanism, so a physical merge would break a load-bearing path.

| Root | Contents | Served / bundled by | Consumed where |
| --- | --- | --- | --- |
| `assets/brand/` | SVG artwork + app-icon master PNG (this folder) | `static.py` (`/brand`); `tauri.conf.json` (`../../assets/brand`) | web masthead, macOS app |
| `assets/source-icons/` | per-source provider PNG icons (`apple-mail.png`, `whatsapp.png`, `whoop.png`, …) | `static.py` (`/source-icons`); `tauri.conf.json` (`../../assets/source-icons`) | web `BrandIcon` (`/source-icons/<slug>.png`) |
| `assets/fonts/` | self-hosted webfonts (`fonts.css` + woff2) | `static.py` (`/fonts`); `tauri.conf.json` (`../../assets/fonts`) | web + macOS type |
| `packages/ui-kit/src/tokens.css` | design tokens (palette + typography), single source | `make tokens` → iOS `apps/estormi-ios/Sources/Design/Tokens.swift` | web (CSS) + iOS (Swift) |
| `apps/estormi-ios/Sources/Resources/Fonts/` | independent static `.ttf` subset (Cinzel + EB Garamond) | XcodeGen auto-detects the directory | iOS app — *not* a woff2 copy, no automated sync |
| `apps/estormi-macos/icons/` | generated iconset / `.icns` | `scripts/generate_icons.sh` from `estormi-app-icon.png` | macOS app icon |

`static.py` is `packages/estormi_server/server/static.py`; `tauri.conf.json`
is `apps/estormi-macos/tauri.conf.json`. The brand palette below mirrors the
`tokens.css` source.


Brand palette (mirrors `tokens.css`):

- Charbon ground: `#1a1f29`
- Encre (deepest): `#0d1117`
- Parchemin text: `#f5f1e8` / `#faf8f4`
- Or ancien (gold): `#c8a96b`
- Or clair (bright gold): `#dcba8a`
- Or sombre (deep gold): `#8a7142`
- Brand burgundy (logo ground): `#6e1f2e`
- Pourpre (error/destructive): `#b82e2e`
- Enluminure (lapis): `#1e3a5f` · Vert sauge (success): `#6b8a5f`

Usage rule: the mark carries its own burgundy ground; keep the live app dark, modern, and quiet. Use the wordmark for the Estormi logo. The blocked illuminated initial is the one lettrine device — the brand `E` in the masthead/icon, the section's own initial on page titles (the old bracket-frame drop cap was retired). Gold is the main accent; red exists only for errors or destructive states. Use the illuminated rule for quiet spacing between major page zones.
